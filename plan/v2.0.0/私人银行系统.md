# 私人银行系统 (Private Banking System) 设计文档

## 1. 概述

本系统是基于 IM 框架的二级经济模型。旨在为高净值用户提供“金融经营”玩法，同时为普通用户提供多样化的存储选择、身份荣誉和信贷支持。

------

## 2. 角色定义

| **角色**             | **准入条件**     | **核心目标**                           |
| -------------------- | ---------------- | -------------------------------------- |
| **主银行 (System)**  | 系统内置         | 调控基础利率，维持经济稳定，发放国卷。 |
| **行长 (Owner)**     | 主银行余额 > 10M | 赚取利差、操作杠杆、建立社交势力。     |
| **储户 (Depositor)** | 无               | 获取利息、赚取专属称号、申请贷款。     |

------

## 3. 私人银行核心机制

### 3.1 准入与创建

- **创建门槛**：主银行余额需达到 **10,000,000 (10M)**。
- **保证金制度**：创建时，10M 将作为**风险保证金**锁定。银行存续期间不可提取，仅在银行倒闭结算时用于偿付储户。
- **自定义权限**：行长可设置银行名称、Logo、Slogan 以及专属 VIP 准入名单。

### 3.2 资金流向逻辑 (80/20 准备金模型)

为了平衡行长收益与储户安全，系统采用自动分配机制：

1. **自动存款**：储户存入的资金，**80%** 自动进入主银行（以私人银行名义存储），享受主银行基准利率。
2. **流动金池**：剩余 **20%** 留在该私人银行的“银行账户”中，用于应对日常取款。
3. **利差收益**：行长设置的“给储户利率”与“主银行基准利率”之间的差额即为行长的被动收入。

------

## 4. 核心功能模块

### 4.1 国卷系统 (Leverage System)

- **定义**：主银行每周限量发行的超高收益理财凭证。
- **竞拍权**：仅限私人银行行长参与。
- **机制**：行长可动用私人银行账户内的资金认购。国卷利率通常为基准利率的 **2-3 倍**，但伴随 3-7 天的**锁仓期**。
- **价值**：这是行长将储户资金转化为个人暴利的核心手段。

### 4.2 称号与荣誉系统

- **自定义称号**：行长可根据银行星级（1-5星）自定义 **1-3 个** 专属称号。
- **佩戴门槛**：行长可设置条件（如：存款满 1M 自动获得【XX行·黄金会员】）。
- **社交价值**：称号在 IM 聊天场景中显示，作为身份象征。

### 4.3 信用与星级评分

- **评分维度**：由存款总额、取款成功率、储户评价（1-5分）综合计算。
- **星级权益**：
  - **高星级**：更多称号位、更高的利率浮动范围、国卷竞拍优先权。
  - **低星级**：功能受限，面临系统监管。

### 4.4 私人信贷 (Loan)

- **发布贷款**：行长可使用自有资金或银行闲置资金发布贷款标的。
- **利息自定**：行长可设置高额利息，针对急需资金购买道具的用户提供短期借贷。

------

## 5. 风险控制与清算 (Risk Management)

### 5.1 强制取款优先级

当储户发起取款请求时，系统按以下顺序强行扣款：

1. **银行流动金池** (20% 留存部分)
2. **银行库存** (行长注入的启动资金)
3. **行长个人钱包**
4. **行长购买的国卷资产** (强制提前折价赎回)
5. **10M 风险保证金**

### 5.2 失信惩罚机制 (Defaulter Management)

- **触发条件**：上述所有渠道均不足以支付储户取款。
- **惩罚措施**：
  - 银行标记为“**失信银行**”。
  - 一个月内利率强制同步主银行，不可调整。
  - 禁止参与国卷竞拍。
  - 行长全服公告通报。
- **恢复**：连续一个月无取款失败记录且库存补足至 10M 后解除。
