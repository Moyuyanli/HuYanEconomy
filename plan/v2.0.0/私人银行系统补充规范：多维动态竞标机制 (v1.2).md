# 私人银行系统补充规范：多维动态竞标机制 (v1.2)

## 1. 竞标模型概述

国卷竞标不再是简单的“随机分配”，而是一场结合了**财力、战略眼光与经营信誉**的博弈。行长通过调整三个核心参数来争取中标机会。

### 1.1 竞标指令

行长通过以下指令参与每月 1 号和 15 号的竞标：
`国卷竞标 (国卷code) (溢价金额) (竞标利率)`

* **溢价金额 (Premium)**：行长愿意额外支付给系统的“入场费”。无论盈亏，中标即扣除并**销毁**。
* **竞标利率 (Bid Interest)**：行长愿意接受的该国卷最终日利率（必须  国卷原始利率）。

---

## 2. 综合中标评分算法 (Winning Score Formula)

系统在截标时（18:00），会对所有投标者计算 **综合得分 (Total Score)**：

### 2.1 维度权重拆解

1. **溢价得分 () - 权重 35%**
* 计算：`你的溢价 / 全场最高溢价`。
* 目的：鼓励货币销毁，溢价越高，系统越倾向于让你中标。


2. **让利得分 () - 权重 25%**
* 计算：`(国卷原始利率 - 你的竞标利率) / 原始利率`。
* 目的：控制通胀。愿意拿更低利息的行长对系统更“友好”。


3. **星级得分 () - 权重 25%**
* 计算：根据银行星级（1-5星）对应固定分值（如：5星=100分，1星=20分）。
* 目的：奖励长期诚信经营、好评度高的银行。


4. **资产规模得分 () - 权重 15%**
* 计算：`B-A 账户余额 / 全场最高 B-A 余额`。
* 目的：确保大额国卷流向资金最雄厚、抗风险能力最强的银行。



---

## 3. 竞标执行流程 (Execution Flow)

### 第一阶段：暗标提交 (08:00 - 18:00)

* **资金检查**：系统检查 `B-A 账户` 余额是否  (国卷面额 + 溢价金额)。
* **预冻结**：提交申请后，系统预冻结 `国卷面额`。溢价金额不预冻结，但若结算时余额不足将视为竞标无效。

### 第二阶段：结算与扣款 (18:05)

1. **判定中标者**：系统选出每个 Code 得分最高的一位或多位（取决于该卷发行份数）。
2. **强制扣费（溢价）**：中标行长的溢价金额立即扣除并从世界货币总量中删除。
3. **资产转化**：`B-A 账户` 中的面额正式转为“国卷资产”。
4. **利率生效**：该国卷在持有期内的实际利率 = **行长填写的竞标利率**。

---

## 4. 三大博弈策略分析

| 策略名称 | 操作核心 | 适用人群 | 优缺点 |
| --- | --- | --- | --- |
| **氪金碾压流** | 极高的溢价金额 | 新晋巨富，星级不高 | 中标率极高，但前期经营成本巨大。 |
| **内卷让利流** | 极低的竞标利率 | 精打细算的稳健行长 | 几乎没有溢价成本，但利润空间被压缩。 |
| **信誉至上流** | 依靠 5 星评级加成 | 老牌口碑银行 | 可以在溢价和利率上保持优势，凭借信誉“降维打击”。 |

---

## 5. 风险与补偿机制

* **流标处理**：若某张国卷无人竞标，该卷自动销毁，下个周期不再补发。
* **利率下限**：行长填写的“竞标利率”不得低于主银行基准利率的均值（防止极端恶意竞争导致系统逻辑错误）。
* **信息确认**：执行指令后，系统必须返回**“当前竞标压力评估”**（例如：*“当前已有 5 人竞标此卷，您的综合竞争力排名：第 2”*），以增强互动感。

---

## 6. 对 IM 系统的宏观意义

1. **货币回收**：通过“溢价金额”精准回收大户手中的过剩货币。
2. **活跃度提升**：双周一次的竞标成为了行长们展示财力与智力的“金融战争”。
3. **阶层流动**：高星级的小银行有机会通过“让利”策略战胜大银行，抢夺优质资产实现阶层跨越。
