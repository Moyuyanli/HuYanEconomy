## 一、指令系统概述

### 触发方式

壶言经济的指令通过 QQ 群消息触发。指令匹配规则由 `@MessageAuthorize` 注解定义，由 `AuthorizeServer` 自动扫描注册。

### 指令前缀

可在配置文件中设置指令前缀（`prefix` 字段）：

- **无前缀**（默认）：直接发送指令关键词即可，如 `签到`
- **有前缀**：需在指令前添加前缀，如设置 `/` 后需发送 `/签到`

### 权限体系

指令权限分为以下层级：

| 权限类型 | 说明 |
|----------|------|
| 无权限要求 | 所有用户均可使用 |
| 自定义权限 | 需要在权限系统中授予对应权限码 |
| 群主/管理员 | 仅群主或管理员可用 |
| 黑名单 | 拥有黑名单权限的用户无法使用 |

## 二、Console 终端命令

在 mirai-console 终端中执行的命令，不在 QQ 群中触发。

### hye v — 查看版本

```
hye v
```

- **说明**：查看当前壶言经济插件版本
- **权限**：Console 终端
- **示例输出**：`当前壶言经济版本 2.0.0`

### hye repair — 数据库修复

```
hye repair
```

- **说明**：修复版本迭代带来的数据库结构变更
- **权限**：Console 终端
- **使用场景**：版本升级后执行，确保数据库表结构与实体类一致

## 三、签到系统

### 签到

**触发词**：`签到` / `打卡` / `sign`

- **说明**：每日签到获取随机金币（50~500）
- **权限**：无（所有用户可用）
- **冷却**：每日一次，刷新时间由配置 `reSignTime` 控制
- **功能**：
  - 随机获得 50~500 金币
  - 支持图片签到（含头像、昵称、金额等信息）
  - 连续签到计数
  - 签到时可能获得随机道具
  - 长期签到可解锁称号（连续签到 15 天、90 天等）

**示例**：
```
用户发送：签到
机器人回复：[签到图片]
  签到成功！获得 328 金币
  连续签到：7 天
```

**签到黑名单**：
- 权限码 `sign-black` 对应的用户无法签到
- 权限组：签到黑名单

## 四、银行系统

### 存款

**触发词**：`存款 <金额>`

- **说明**：将钱包中的金币存入主银行
- **权限**：无
- **参数**：
  - `<金额>`（必填）：存款金额，正整数
- **示例**：
  ```
  用户发送：存款 1000
  机器人回复：存款成功！存入 1000 金币
  ```

### 一键存款

**触发词**：`存款!`

- **说明**：将钱包中的全部金币存入主银行
- **权限**：无
- **示例**：
  ```
  用户发送：存款!
  机器人回复：一键存款成功！存入 3580 金币
  ```

### 取款

**触发词**：`取款 <金额>`

- **说明**：从主银行取出金币到钱包
- **权限**：无
- **参数**：
  - `<金额>`（必填）：取款金额，正整数
- **示例**：
  ```
  用户发送：取款 500
  机器人回复：取款成功！取出 500 金币
  ```

### 查看利率

**触发词**：`银行利率`

- **说明**：查看当前主银行利率
- **权限**：无

### 富豪榜

**触发词**：`富豪榜`

- **说明**：查看群内财富排行榜，包含钱包余额、银行存款及占比
- **权限**：无

## 五、私人银行系统

### 创建银行

**触发词**：`银行创建 <code> <name>`

- **说明**：创建一家私人银行
- **权限**：无（但需满足条件）
- **条件**：主银行存款 ≥ 1 亿金币，需缴纳 1000 万保证金
- **参数**：
  - `<code>`（必填）：银行唯一编码（建议 `pb-` 前缀）
  - `<name>`（必填）：银行名称
- **示例**：
  ```
  用户发送：银行创建 pb-test 测试银行
  机器人回复：银行创建成功！编码：pb-test，名称：测试银行
  ```

### 银行列表

**触发词**：`银行列表`

- **说明**：查看所有私人银行列表
- **权限**：无

### 银行信息

**触发词**：`银行信息 [code]`

- **说明**：查看指定银行的详细信息；不填时优先查看自己创建的银行，其次查看默认银行
- **权限**：无
- **参数**：
  - `[code]`（可选）：银行编码，不填则查看自己创建的银行或默认银行

### 默认银行

**触发词**：`默认银行`

- **说明**：查询当前默认银行；未配置时默认为主银行
- **权限**：无

**触发词**：`默认银行设置 <code/name|main|主银行>`

- **说明**：主动切换默认银行；填 `main` 或 `主银行` 时切回主银行
- **权限**：无
- **参数**：
  - `<code/name|main|主银行>`（必填）：银行编码、银行名称，或主银行别名

### 向私人银行存款

**触发词**：`存款 <金额> [code]`

- **说明**：向指定私人银行存款
- **权限**：无
- **参数**：
  - `<金额>`（必填）：存款金额
  - `[code]`（可选）：银行编码，不填则使用默认银行

### 向私人银行取款

**触发词**：`取款 <金额> [code]`

- **说明**：从指定私人银行取款
- **权限**：无

### 贷款

**触发词**：`贷款 <金额> [code]`

- **说明**：向指定私人银行申请贷款
- **权限**：无
- **参数**：
  - `<金额>`（必填）：贷款金额
  - `[code]`（可选）：银行编码

### 还款

**触发词**：`还款 <金额> [code]`

- **说明**：向指定私人银行还款
- **权限**：无

### 银行评分

**触发词**：`银行评分 <1-5> [描述] [code]`

- **说明**：给指定银行评分（1-5 星）
- **权限**：无
- **参数**：
  - `<1-5>`（必填）：评分，1-5 的整数
  - `[描述]`（可选）：评价文字
  - `[code]`（可选）：银行编码

### 行长放贷

**触发词**：`放贷 <金额> [利率]` / `银行放贷 <金额> [利率]`

- **说明**：行长发布放贷标的，供他人贷款
- **权限**：仅银行行长
- **参数**：
  - `<金额>`（必填）：放贷金额，支持 `100000`、`20K`、`10M`、`1亿` 等写法
  - `[利率]`（可选）：日利率，不填则使用该银行当前存款利率

### 狐卷相关

**触发词**：`狐卷`

- **说明**：查看当前狐卷信息
- **权限**：无

**触发词**：`狐卷竞标 <code> <溢价> <利率>`

- **说明**：行长参与狐卷竞标
- **权限**：仅银行行长
- **参数**：
  - `<code>`（必填）：狐卷编码
  - `<溢价>`（必填）：溢价金额，支持 `20K`、`10M` 等写法
  - `<利率>`（必填）：接受的日利率

### 国卷相关

**触发词**：`国卷` / `国卷查看` / `国卷列表`

- **说明**：查看本周国卷发行信息，以及自己银行的国卷持仓
- **权限**：无

**触发词**：`国卷购买 <金额>`

- **说明**：行长使用银行流动金池购买本周国卷
- **权限**：仅银行行长
- **参数**：
  - `<金额>`（必填）：购买金额，支持 `20K`、`10M`、`1亿` 等写法

**触发词**：`国卷赎回 [持仓ID]`

- **说明**：赎回指定国卷持仓；不填 ID 时尝试赎回全部未赎回持仓
- **权限**：仅银行行长

## 六、钓鱼系统

### 购买鱼竿

**触发词**：`购买鱼竿`

- **说明**：购买钓鱼竿（500 金币）
- **权限**：`fish`（钓鱼权限）
- **前置条件**：金币 ≥ 500，且未拥有鱼竿

### 钓鱼

**触发词**：`钓鱼` / `抛竿`

- **说明**：开始钓鱼，进入操作博弈流程
- **权限**：`fish`（钓鱼权限）
- **前置条件**：已购买鱼竿
- **流程**：
  1. 发送钓鱼指令
  2. 系统随机选择鱼塘中的鱼
  3. 进入操作博弈（新版模式）
  4. 根据操作结果决定是否钓到鱼
  5. 鱼的尺寸和价值由随机因子决定

### 升级鱼竿

**触发词**：`升级鱼竿`

- **说明**：升级鱼竿等级，提升钓鱼能力
- **权限**：`fish`（钓鱼权限）
- **前置条件**：已购买鱼竿，不在钓鱼中

### 查看鱼竿等级

**触发词**：`鱼竿等级`

- **说明**：查看当前鱼竿等级
- **权限**：无

### 查看鱼塘等级

**触发词**：`鱼塘等级`

- **说明**：查看当前鱼塘等级和信息
- **权限**：无

### 钓鱼排行榜

**触发词**：`钓鱼排行榜`

- **说明**：查看钓鱼排行榜（按鱼的尺寸或价值排名）
- **权限**：无

## 七、抢劫系统

### 抢劫玩家

**触发词**：`抢劫 @用户`

- **说明**：抢劫其他群成员的金币
- **权限**：`rob`（抢劫权限）
- **冷却**：由配置 `robCoolTime` 控制（默认 120 秒）
- **机制**：
  - 成功：获得目标部分金币（上限由 `robMaxMoney` 控制）
  - 失败：可能被罚款或被抓入监狱
  - 入狱：被禁言，时长由 `jailCoolTime` 控制（默认 3600 秒）

### 抢银行

**触发词**：`抢银行`

- **说明**：尝试抢劫银行（成功率极低）
- **权限**：`rob`（抢劫权限）
- **基础概率**：1%（由 `GlobalFactor.robBlankFactor` 控制）

### 释放出狱

**触发词**：`释放出狱`

- **说明**：提前出狱（如有保释人或其他条件）
- **权限**：无（仅在押状态可用）

### 保释他人

**触发词**：`保释 @用户`

- **说明**：保释被关押的其他玩家
- **权限**：无

## 八、红包系统

### 发红包

**触发词**：`发红包 <金额> <个数> [随机]`

- **说明**：在群内发放红包
- **权限**：`red-pack`（红包权限）
- **参数**：
  - `<金额>`（必填）：红包总金额
  - `<个数>`（必填）：红包个数
  - `[随机]`（可选）：添加此参数为随机红包，不填则为普通红包（平均分配）
- **示例**：
  ```
  用户发送：发红包 100 5
  机器人回复：[红包消息] 发出 100 金币的红包，共 5 个

  用户发送：发红包 100 5 随机
  机器人回复：[红包消息] 发出 100 金币的随机红包，共 5 个
  ```

### 领取红包

**触发词**：`抢红包` / `领红包 <id>`

- **说明**：领取群内的红包
- **权限**：无
- **参数**：
  - `<id>`（可选）：红包 ID，不填则领取最新的

### 红包列表

**触发词**：`红包列表`

- **说明**：查看当前群内可领取的红包列表
- **权限**：无

**红包规则**：
- 红包 24 小时未领完自动退回发送者
- 每人每红包只能领取一次
- 口令红包需输入正确口令才能领取
