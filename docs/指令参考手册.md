## 一、指令系统概述

### 触发方式

壶言经济的指令通过 QQ 群消息触发。指令匹配规则由 `@MessageAuthorize` 注解定义，由 `AuthorizeServer` 自动扫描注册。

### 指令前缀

可在配置文件中设置指令前缀（`prefix` 字段）：

- **无前缀**（默认）：直接发送指令关键词即可，如 `签到`
- **有前缀**：需在指令前添加前缀，如设置 `/` 后需发送 `/签到`

### 权限体系

指令权限分为以下层级：

| 权限类型 | 说明 |
|----------|------|
| 无权限要求 | 所有用户均可使用 |
| 自定义权限 | 需要在权限系统中授予对应权限码 |
| 群主/管理员 | 仅群主或管理员可用 |
| 黑名单 | 拥有黑名单权限的用户无法使用 |

## 二、Console 终端命令

在 mirai-console 终端中执行的命令，不在 QQ 群中触发。

### hye v — 查看版本

```
hye v
```

- **说明**：查看当前壶言经济插件版本
- **权限**：Console 终端
- **示例输出**：`当前壶言经济版本 2.0.0`

### hye repair — 数据库修复

```
hye repair
hye repair v1
hye repair v2
hye repair V1TOV2
```

- **说明**：无参数时仅显示帮助，不修改数据库。`v1` 仅修复 V1 旧表；`v2` 仅修复 V2 新表；`V1TOV2` 将 V1 作为只读备份源，修复迁移到 V2 时丢失的数据
- **权限**：Console 终端
- **使用场景**：根据数据所在版本选择范围。已全面切换 V2 且需保留 V1 备份时，只使用 `v2` 或 `V1TOV2`

### hye repair privatebank ledger — 清理孤儿私人银行账本

```
hye repair privatebank ledger <code>
```

- **说明**：清理没有私人银行实体、没有存款/贷款/债券等业务记录的历史私人银行资金账本
- **权限**：Console 终端
- **使用场景**：旧历史资金残留导致新建银行 code 被拦截，且确认该 code 已无有效银行数据时使用

### hye repair privatebank — 审计与修复在用银行

```
hye repair privatebank audit <code>
hye repair privatebank deposit <code> <qq> <correctPrincipal> confirm
hye repair privatebank reconcile <code> confirm
```

- **说明**：`audit` 只读检查银行营业/破产状态、三池、放贷标的和主银行债务；`deposit` 将指定储户本金校正到明确值且仅允许下调；`reconcile` 修复负放贷库存并同步标的剩余额度
- **权限**：Console 终端
- **事故恢复顺序**：先 `audit`，再校正已知错误的储户本金，随后 `reconcile`，最后再次 `audit`
- **安全约束**：两个写命令必须以 `confirm` 结尾，且重复执行不会重复扣减
- **破产限制**：破产银行仅允许执行 `audit`，其他修复和全部用户业务永久关闭

## 三、签到系统

### 签到

**触发词**：`签到` / `打卡` / `sign`

- **说明**：每日签到获取随机金币（50~500）
- **权限**：无（所有用户可用）
- **冷却**：每日一次，刷新时间由配置 `reSignTime` 控制
- **功能**：
  - 随机获得 50~500 金币
  - 支持图片签到（含头像、昵称、金额等信息）
  - 连续签到计数
  - 签到时可能获得随机道具
  - 长期签到可解锁称号（连续签到 15 天、90 天等）

**示例**：
```
用户发送：签到
机器人回复：[签到图片]
  签到成功！获得 328 金币
  连续签到：7 天
```

**签到黑名单**：
- 权限码 `sign-black` 对应的用户无法签到
- 权限组：签到黑名单

## 四、银行系统

### 存款

**触发词**：`存款 <金额>`

- **说明**：将钱包中的金币存入默认银行；未配置默认银行或默认银行为主银行时存入主银行
- **权限**：无
- **参数**：
  - `<金额>`（必填）：存款金额，正整数
- **示例**：
  ```
  用户发送：存款 1000
  机器人回复：存款成功！存入 1000 金币
  ```

### 一键存款

**触发词**：`存款!`

- **说明**：将钱包中的全部金币存入默认银行；未设置默认银行时存入主银行
- **权限**：无
- **示例**：
  ```
  用户发送：存款!
  机器人回复：一键存款成功！存入 3580 金币
  ```

### 取款

**触发词**：`取款 <金额>`

- **说明**：从默认银行取出金币到钱包；未配置默认银行或默认银行为主银行时从主银行取款
- **权限**：无
- **参数**：
  - `<金额>`（必填）：取款金额，正整数
- **示例**：
  ```
  用户发送：取款 500
  机器人回复：取款成功！取出 500 金币
  ```

### 查看利率

**触发词**：`银行利率`

- **说明**：查看当前主银行利率
- **权限**：无

### 富豪榜

**触发词**：`富豪榜`

- **说明**：查看群内财富排行榜，包含钱包余额、银行存款及占比
- **权限**：无

## 五、私人银行系统

### 创建银行

**触发词**：`银行创建 <code> <name>`

- **说明**：创建一家私人银行
- **权限**：无（但需满足条件）
- **条件**：主银行存款 ≥ 1 亿金币，需缴纳 1000 万保证金
- **参数**：
  - `<code>`（必填）：银行唯一编码（建议 `pb-` 前缀）
  - `<name>`（必填）：银行名称
- **示例**：
  ```
  用户发送：银行创建 pb-test 测试银行
  机器人回复：银行创建成功！编码：pb-test，名称：测试银行
  ```

### 银行列表

**触发词**：`银行列表`

- **说明**：查看所有私人银行列表
- **权限**：无

### 银行信息

**触发词**：`银行信息 [code]`

- **说明**：查看指定银行的详细信息；不填时优先查看自己创建的银行，其次查看默认银行
- **权限**：无
- **参数**：
  - `[code]`（可选）：银行编码，不填则查看自己创建的银行或默认银行

### 我的银行

**触发词**：`我的银行` / `我的银行#`

- **说明**：查看自己创建的银行信息；`我的银行` 返回图片，`我的银行#` 返回文字
- **权限**：无
- **备注**：未创建银行时返回 `你没有创建银行`

### 默认银行

**触发词**：`默认银行`

- **说明**：查询当前默认银行；未配置时默认为主银行
- **权限**：无

**触发词**：`默认银行设置 <code/name|main|主银行>`

- **说明**：主动切换默认银行；填 `main` 或 `主银行` 时切回主银行
- **权限**：无
- **参数**：
  - `<code/name|main|主银行>`（必填）：银行编码、银行名称，或主银行别名

### 向私人银行存款

**触发词**：`存款 <金额> [code]`

- **说明**：带 `code` 时向指定私人银行存款；不带 `code` 时优先使用默认银行
- **权限**：无
- **参数**：
  - `<金额>`（必填）：存款金额
  - `[code]`（可选）：银行编码；填写 `main` 或 `主银行` 时强制使用主银行

### 向私人银行取款

**触发词**：`取款 <金额> [code]`

- **说明**：带 `code` 时从指定私人银行取款；不带 `code` 时优先使用默认银行；填写 `main` 或 `主银行` 时强制使用主银行
- **权限**：无

### 银行补资

**触发词**：`银行补资 <金额> [P|F|PF|FP]`

- **说明**：行长向自己的银行补充资金；`P` 为准备金池，`F` 为流动金池，省略时默认 `P`
- **资金来源**：先检查钱包，钱包可单独支付时只扣钱包；否则检查主银行，主银行可单独支付时只扣主银行；两个账户余额不合并计算
- **池间转移**：两个字母按“来源池 → 目标池”解释；`PF` 表示准备金池转入流动金池，`FP` 表示流动金池转入准备金池
- **权限**：仅银行行长

### 贷款

**触发词**：`贷款 <金额> [code]`

- **说明**：向指定私人银行申请贷款
- **权限**：无
- **参数**：
  - `<金额>`（必填）：贷款金额
  - `[code]`（可选）：银行编码

### 还款

**触发词**：`还款 <金额> [code]`

- **说明**：向指定私人银行还款
- **权限**：无

### 银行评分

**触发词**：`银行评分 <1-5> [描述] [code]`

- **说明**：给指定银行评分（1-5 星）
- **权限**：无
- **参数**：
  - `<1-5>`（必填）：评分，1-5 的整数
  - `[描述]`（可选）：评价文字
  - `[code]`（可选）：银行编码

### 行长放贷

**触发词**：`放贷 <金额> [利率]` / `银行放贷 <金额> [利率]`

- **说明**：行长发布放贷标的，供他人贷款
- **权限**：仅银行行长
- **参数**：
  - `<金额>`（必填）：放贷金额，支持 `100000`、`20K`、`10M`、`1亿` 等写法
  - `[利率]`（可选）：日利率，范围 1.0% - 18.0%；不填则使用该银行当前存款利率

### 修改贷款利息

**触发词**：`贷款利息修改 <offerId> <利率>` / `放贷利息修改 <offerId> <利率>`

- **说明**：修改尚可用贷款额度的日利率
- **权限**：仅银行行长
- **参数**：
  - `<offerId>`（必填）：贷款额度 ID
  - `<利率>`（必填）：日利率，范围 1.0% - 18.0%

### 贷款额度管理

**触发词**：`放贷列表`

- **说明**：查看自己银行当前有效的贷款额度、剩余额度、日利率和状态；每个额度下附带尚未结清的借款人、借款本金和利息
- **过滤规则**：已结清或追缴完成的借款、已撤回的贷款额度不展示且不参与汇总，仅保留数据库记录
- **权限**：仅银行行长

### 借贷列表

**触发词**：`借贷列表`

- **说明**：查看自己从各私人银行借入的本金、日利率、利息、已还/待还金额和最迟还款时间
- **权限**：无

**触发词**：`撤贷 <offerId>` / `贷款撤回 <offerId>` / `放贷撤回 <offerId>`

- **说明**：撤回尚未借出的贷款额度；已借出的借款单继续保留
- **权限**：仅银行行长
- **参数**：
  - `<offerId>`（必填）：贷款额度 ID

### 狐卷相关

**触发词**：`狐卷` / `狐卷查看` / `狐卷 查看` / `狐卷列表`

- **说明**：查看当前仍在竞标时间内的狐卷信息
- **权限**：无

**触发词**：`狐卷竞标 <code> <溢价> <利率>`

- **说明**：行长参与狐卷竞标
- **权限**：仅银行行长
- **参数**：
  - `<code>`（必填）：狐卷编码
  - `<溢价>`（必填）：溢价金额，支持 `20K`、`10M` 等写法
  - `<利率>`（必填）：接受的日利率

**触发词**：`狐卷补发`

- **说明**：立即补发一批 10 张狐卷，编号续接当日已有狐卷；从补发时刻起开放 10 小时竞标
- **权限**：仅 `owner`

### 国卷相关

**触发词**：`国卷` / `国卷列表`

- **说明**：以转发消息分页展示当前仍可购买的国卷，每页最多 15 个
- **展示规则**：仅展示剩余额度大于 0，且距离发行赎回时间超过 24 小时的国卷
- **权限**：无

**触发词**：`国卷持仓`

- **说明**：以转发消息展示发送者作为行长所拥有银行的未赎回国卷持仓，包括本金、利率、购买时间、赎回时间和状态
- **权限**：仅银行行长

**触发词**：`购买国卷 <金额> <code>`

- **说明**：行长使用银行流动金池购买指定国卷；已过期或距离发行赎回不足 24 小时的国卷不可购买
- **权限**：仅银行行长
- **持仓期限**：购买后的实际赎回时间按“购买时间 + 国卷锁仓天数”计算
- **参数**：
  - `<金额>`（必填）：购买金额，支持 `20K`、`10M`、`1亿` 等写法
  - `<code>`（必填）：国卷编码，例如 `GB-20260714-001`

**触发词**：`赎回国卷 <code> [金额]`

- **说明**：赎回指定国卷编码的持仓；不填金额时赎回该编码下的全部可赎回本金
- **权限**：仅银行行长
- **参数**：
  - `<code>`（必填）：国卷编码
  - `[金额]`（可选）：指定赎回本金，支持金额缩写

**触发词**：`国卷补发`

- **说明**：立即补发一批国卷，数量按正常发行权重生成，编号续接当日已有国卷
- **权限**：仅 `owner`

## 六、钓鱼系统

### 购买鱼竿

**触发词**：`购买鱼竿`

- **说明**：购买钓鱼竿（500 金币）
- **权限**：`fish`（钓鱼权限）
- **前置条件**：金币 ≥ 500，且未拥有鱼竿

### 钓鱼

**触发词**：`钓鱼` / `抛竿`

- **说明**：开始钓鱼，进入操作博弈流程
- **权限**：`fish`（钓鱼权限）
- **前置条件**：已购买鱼竿
- **流程**：
  1. 发送钓鱼指令
  2. 系统随机选择鱼塘中的鱼
  3. 进入操作博弈（新版模式）
  4. 根据操作结果决定是否钓到鱼
  5. 鱼的尺寸和价值由随机因子决定

### 升级鱼竿

**触发词**：`升级鱼竿`

- **说明**：升级鱼竿等级，提升钓鱼能力
- **权限**：`fish`（钓鱼权限）
- **前置条件**：已购买鱼竿，不在钓鱼中

### 查看鱼竿等级

**触发词**：`鱼竿等级`

- **说明**：查看当前鱼竿等级
- **权限**：无

### 查看鱼塘等级

**触发词**：`鱼塘等级`

- **说明**：查看当前鱼塘等级和信息
- **权限**：无

### 钓鱼排行榜

**触发词**：`钓鱼排行榜`

- **说明**：查看钓鱼排行榜（按鱼的尺寸或价值排名）
- **权限**：无

- **权限**：`fish`（钓鱼权限）

## 七、农场系统

### 我的农场

**触发词**：`我的农场` / `农场信息`

- **说明**：查看农场详情图片
- **权限**：`farm`（农场权限）

**触发词**：`我的农场#` / `农场信息#`

- **说明**：查看农场文字概览
- **权限**：`farm`（农场权限）

### 农场互动

**触发词**：`帮浇水 @用户 [*次数]`

- **说明**：为其他玩家尚未成熟的作物缩短成熟时间，有概率获得农场抽奖券
- **解锁**：农场 13 级

**触发词**：`偷菜 @用户`

- **说明**：随机尝试偷取目标一块已经成熟、当前季尚未被偷的 12 级及以上作物
- **次数**：与帮浇水共享每日次数；13–17 级每日 5 次，18 级起每日 10 次
- **扣次**：选中有效作物后即扣除一次，概率失败或被守护拦截同样扣除
- **奖励**：成功获得果实并使主人当季减产；偷菜不会掉落农场抽奖券

### 农场守护

**触发词**：`激活守护`

- **解锁**：农场 17 级
- **效果**：持续 12 小时，期间所有偷菜尝试直接失败
- **费用**：当前等级对应升级消耗的 5%，向上取整
- **限制**：守护生效期间不能重复激活，也不会重复扣费

## 八、抢劫系统

### 抢劫玩家

**触发词**：`抢劫 @用户`

- **说明**：抢劫其他群成员的金币
- **权限**：`rob`（抢劫权限）
- **冷却**：由配置 `robCoolTime` 控制（默认 120 秒）
- **机制**：
  - 成功：获得目标部分金币（上限由 `robMaxMoney` 控制）
  - 失败：可能被罚款或被抓入监狱
  - 入狱：被禁言，时长由 `jailCoolTime` 控制（默认 3600 秒）

### 抢银行

**触发词**：`抢银行`

- **说明**：尝试抢劫银行（成功率极低）
- **权限**：`rob`（抢劫权限）
- **基础概率**：1%（由 `GlobalFactor.robBlankFactor` 控制）

### 释放出狱

**触发词**：`释放出狱`

- **说明**：提前出狱（如有保释人或其他条件）
- **权限**：无（仅在押状态可用）

### 保释他人

**触发词**：`保释 @用户`

- **说明**：保释被关押的其他玩家
- **权限**：无

## 九、红包系统

### 发红包

**触发词**：`发红包 <金额> <个数> [随机]`

- **说明**：在群内发放红包
- **权限**：`red-pack`（红包权限）
- **参数**：
  - `<金额>`（必填）：红包总金额
  - `<个数>`（必填）：红包个数
  - `[随机]`（可选）：添加此参数为随机红包，不填则为普通红包（平均分配）
- **示例**：
  ```
  用户发送：发红包 100 5
  机器人回复：[红包消息] 发出 100 金币的红包，共 5 个

  用户发送：发红包 100 5 随机
  机器人回复：[红包消息] 发出 100 金币的随机红包，共 5 个
  ```

**触发词**：`抢红包` / `领红包 <id>`

- **说明**：领取群内的红包
- **权限**：无
- **参数**：
  - `<id>`（可选）：红包 ID，不填则领取最新的

### 红包列表

**触发词**：`红包列表`

- **说明**：查看当前群内可领取的红包列表
- **权限**：无

**红包规则**：
