## 一、配置文件概述

壶言经济的配置文件位于 Mirai Console 的 `config/` 目录下，分为两个插件目录：

```
config/
├── cn.chahuyun.HuYanEconomy/        # 壶言经济配置
│   ├── Config.yml                   # 主配置
│   ├── PluginConfig.yml             # 插件运行配置
│   ├── FishingMsgConfig.yml         # 钓鱼消息文案
│   ├── RobMsgConfig.yml             # 抢劫消息文案
│   └── PrizesData.yml               # 奖品数据
└── cn.chahuyun.HuYanAuthorize/      # 权限系统配置
    └── AuthorizeConfig.yml          # 权限配置
```

> **提示**：首次启动插件后立即停止，即可自动生成默认配置文件。所有配置类基于 mirai-console 的 `AutoSavePluginConfig`，支持热重载。

## 二、主配置 Config.yml

**对应类**：`cn.chahuyun.economy.config.EconomyConfig`

这是壶言经济的核心配置文件，包含数据库连接、Bot 绑定、签到规则等基础设置。

### 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `owner` | Long | `123456` | 主人 QQ 号，拥有最高管理权限，可使用 `greedisgood` 等管理员指令 |
| `prefix` | String | `""`（空字符串） | 指令触发前缀。设置后所有指令需此前缀才能触发，如设置为 `/` 则签到变为 `/签到` |
| `reSignTime` | Int | `4` | 每日签到刷新时间（0-23 时），超过此时间后签到状态重置 |
| `dataType` | DriveType | `H2` | 数据库类型，可选值：`H2`、`SQLITE`、`MYSQL` |
| `mysqlUrl` | String | `localhost:3306/test` | MySQL 连接地址，仅 `dataType=MYSQL` 时生效 |
| `mysqlUser` | String | `root` | MySQL 用户名，仅 `dataType=MYSQL` 时生效 |
| `mysqlPassword` | String | `123456` | MySQL 密码，仅 `dataType=MYSQL` 时生效 |
| `nextMessageExecutorsNumber` | Int | `10` | 用户下一条消息等待线程数，可能影响同时钓鱼的玩家数量 |
| `fishType` | String | `new` | 钓鱼模式。`new` = 新版操作博弈模式，`old` = 旧版简单模式 |
| `unableToUseMuteGroup` | MutableList\<Long\> | `[]` | 无法使用禁言卡的群号列表，列表中的群禁止使用禁言卡道具 |

### 完整示例

```yaml
owner: 123456
prefix: ''
reSignTime: 4
dataType: H2
mysqlUrl: 'localhost:3306/huyan_economy'
mysqlUser: 'root'
mysqlPassword: 'your_password_here'
nextMessageExecutorsNumber: 10
fishType: 'new'
unableToUseMuteGroup:
  - 123456789
  - 987654321
```

## 四、钓鱼消息配置 FishingMsgConfig.yml

**对应类**：`cn.chahuyun.economy.config.FishingMsgConfig`

自定义钓鱼系统中的提示消息文案。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `noneRodMsg` | `"你还没有鱼竿，发送\"购买鱼竿\"购买吧"` | 没有鱼竿时尝试钓鱼的提示 |
| `fishingNowMsg` | `"你已经在钓鱼了！"` | 正在钓鱼时重复尝试的提示 |
| `rodLevelNotEnough` | `"你的鱼竿太拉了，这里不让你来，升升级吧..."` | 鱼竿等级不够时的提示 |
| `repeatPurchaseRod` | `"你已经有一把钓鱼竿了，不用再买了！"` | 已有鱼竿时再次购买的提示 |
| `coinNotEnoughForRod` | `"我这把钓鱼竿可是神器！你的金币还远远不够！等攒够了500金币再来吧"` | 金币不足购买鱼竿的提示 |
| `buyFishingRodSuccess` | `"拿好了，这鱼竿到手即不负责，永不提供售后！"` | 购买鱼竿成功的提示 |
| `noneRodUpgradeMsg` | `"鱼竿都没有，无法升级!"` | 没有鱼竿时尝试升级的提示 |
| `upgradeWhenFishing` | `"钓鱼期间不可升级鱼竿!"` | 钓鱼期间尝试升级的提示 |

### 自定义示例

```yaml
noneRodMsg: "你还没有鱼竿呢~快发送\"购买鱼竿\"买一把吧！"
buyFishingRodSuccess: "恭喜你获得了传说中的鱼竿！祝你钓到大鱼~"
```

## 五、抢劫消息配置 RobMsgConfig.yml

**对应类**：`cn.chahuyun.economy.config.RobMsgConfig`

自定义抢劫系统的参数和消息文案。

### 参数配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `robCoolTime` | Long | `120` | 抢劫冷却时长（秒），两次抢劫之间的等待时间 |
| `jailCoolTime` | Long | `3600` | 入狱被禁时长（秒），被关进监狱后的等待时间 |
| `robMaxMoney` | Int | `1000` | 抢夺受害者最大金钱数量（单次抢劫上限） |

### 消息模板

消息模板中支持以下变量占位符：
- `${对象}` — 被抢劫目标的昵称
- `${金币}` — 涉及的金币数量
- `${时间}` — 相关时间信息

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `robSuccessMsg` | `["你抢到了 ${对象} ${金币} 枚金币"]` | 抢劫成功的提示（随机选择一条） |
| `robFailMsg` | `["你在抢 ${对象} 的金币时被发现了!"]` | 抢劫失败的提示 |
| `loseMoneyMsg` | `["你在抢 ${对象} 的金币时被抓住了!因此赔了 ${金币} 元"]` | 抢劫失败被罚款的提示 |
| `robJailMsg` | `["你在抢 ${对象} 的金币时被警察抓到了!你被关进了监狱，并罚款 ${金币} 元!CD延长至: ${时间}"]` | 被抓入狱的提示 |
| `robNotEnoughMsg` | `"你没钱了！赔不起捏"` | 金币不足无法赔偿的提示 |

### 自定义示例

```yaml
robCoolTime: 180
jailCoolTime: 7200
robMaxMoney: 2000
robSuccessMsg:
  - "你偷偷摸摸地从 ${对象} 口袋里摸走了 ${金币} 金币！"
  - "恭喜！你成功抢走了 ${对象} 的 ${金币} 金币！"
robFailMsg:
  - "糟糕！${对象} 发现了你的企图！"
  - "你被 ${对象} 一脚踹飞了！"
```

## 七、权限配置 HuYanAuthorize

**配置文件**：`config/cn.chahuyun.HuYanAuthorize/AuthorizeConfig.yml`

壶言经济依赖的权限系统配置。

| 配置项 | 说明 |
|--------|------|
| `owner` | 主人 QQ 号（全局管理员） |

| `dataType` | 权限数据库类型（DriveType 枚举，可选值：H2 / SQLITE / MYSQL） |

| 权限码 | 说明 | 权限组 |

| `fish` | 钓鱼权限 | 钓鱼组 |
| `lottery` | 猜签权限 | 猜签组 |
| `rob` | 抢劫权限 | 抢劫组 |
| `red-pack` | 红包权限 | 红包组 |
| `sign-black` | 签到黑名单 | 签到黑名单 |

## 八、数据库配置

### 数据库类型选择

| 数据库 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| **H2**（默认） | 零配置、嵌入式、启动快 | 不支持远程访问、数据量大时性能下降 | 开发测试、小型部署 |
| **SQLite** | 零配置、文件存储、跨平台 | 并发写入性能差 | 小型部署、单机使用 |
| **MySQL** | 高并发、远程访问、成熟稳定 | 需要额外安装配置 | 生产环境、多实例部署 |

### MySQL 配置示例

```yaml
dataType: MYSQL
mysqlUrl: '192.168.1.100:3306/huyan_economy'
mysqlUser: 'huyan_user'
mysqlPassword: 'secure_password_here'
```

### 数据库表管理

- 插件启动时自动建表（Hibernate 自动 DDL）
- 实体类变更后执行 `hye repair` 查看并选择对应的数据版本修复范围
- 数据库文件位置（H2/SQLite）：`data/cn.chahuyun.HuYanEconomy/`

```yaml
# 此时签到变为 /签到，存款变为 /存款
```
