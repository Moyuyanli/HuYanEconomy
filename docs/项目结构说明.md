# 壶言经济（HuYanEconomy）项目结构说明

## 一、项目概述

壶言经济（HuYanEconomy）是一款基于 [Mirai Console](https://github.com/mamoe/mirai) 的 QQ 机器人经济插件，使用 Kotlin 编写。提供签到、银行、钓鱼、抢劫、抽奖、道具、私人银行等娱乐经济功能。

### 技术栈

| 项目 | 技术选型 |
|------|----------|
| 语言 | Kotlin 1.9.20, JVM Target 11 |
| 框架 | mirai-console 2.16.0 |
| 构建 | Gradle KTS |
| ORM | Hibernate（通过 mirai-hibernate-plugin 运行时提供） |
| 权限系统 | [HuYanAuthorize](https://github.com/Moyuyanli/HuYanAuthorize)（KSP 注解处理） |
| 序列化 | kotlinx-serialization |
| 图片生成 | Apache POI + Java AWT |

### 调用流程示例

```mermaid
sequenceDiagram
    participant User as QQ 用户
    participant Auth as AuthorizeServer
    participant Action as Action（控制层）
    participant Usecase as Usecase（业务层）
    participant Manager as Manager（管理层）
    participant Repo as Repository（数据访问层）
    participant Entity as Entity（实体层）

    User->>Auth: 发送消息
    Auth->>Action: @MessageAuthorize 匹配
    Action->>Usecase: 调用业务方法
    Usecase->>Manager: 调用管理器 API
    Manager->>Repo: 查询/持久化数据
    Repo->>Entity: Hibernate ORM
    Entity-->>Repo: 返回实体
    Repo-->>Manager: 返回数据
    Manager-->>Usecase: 返回结果
    Usecase-->>Action: 返回处理结果
    Action-->>User: 回复消息
```

## 五、关键文件说明

### 5.1 入口类 `HuYanEconomy.kt`

插件主类，Kotlin `object` 单例，继承 `KotlinPlugin`。定义插件元信息（ID、版本、依赖），并编排整个插件的初始化流程。

**插件元信息：**
- **ID**: `cn.chahuyun.HuYanEconomy`
- **版本**: 由 `EconomyBuildConstants.VERSION`（KSP 构建时生成）提供
- **依赖**: `mirai-economy-core >= 1.0.6`、`HuYanAuthorize >= 1.3.6`

### 5.2 构建脚本 `build.gradle.kts`

Gradle KTS 构建配置，包含：
- Kotlin 编译器选项与 JVM Target
- 依赖声明（mirai-console、Hibernate 等）
- KSP 注解处理配置（HuYanAuthorize）
- `buildPlugin` 任务配置
- 排除 log4j 防止 ClassLoader 冲突

### 5.3 调试环境 `debug-sandbox/`

本地开发调试用目录，模拟 mirai-console 运行环境：
- `bots/` — Bot QQ 号配置
- `config/` — 各插件运行时配置
- `data/` — 插件运行时数据
- `plugins/` — 依赖插件 jar

### 5.4 KSP 生成代码 `build/generated/ksp/`

由 HuYanAuthorize 的 KSP 处理器自动生成的 `AuthorizeRegistrar`，负责注册 `@MessageAuthorize` 注解的事件处理器。**严禁手动修改。**

```mermaid

        [*] --> 设置插件状态: PLUGIN_STATUS = true
    }

    state onEnable {
        [*] --> 加载配置
        加载配置 --> 注册Console命令
        注册Console命令 --> 初始化工具类
        初始化工具类 --> 注册权限码
        注册权限码 --> 初始化数据库
        初始化数据库 --> 初始化各Manager
        初始化各Manager --> 注册消息监听
        注册消息监听 --> 注册事件监听
        注册事件监听 --> [*]
    }

    state onDisable {
        [*] --> 关闭GamesManager
        关闭GamesManager --> 关闭LotteryManager
        关闭LotteryManager --> 关闭YiYanManager
        关闭YiYanManager --> 停止调度器
        停止调度器 --> [*]
    }
```

### 6.1 `onLoad` 阶段

- 设置 `PLUGIN_STATUS = true`

### 6.2 `onEnable` 阶段

按以下顺序初始化：

1. **加载配置** — `EconomyConfig`、`EconomyPluginConfig`、`FishingMsgConfig`、`RobMsgConfig`、`PrizesData`
2. **注册 Console 命令** — `EconomyCommand`
3. **初始化工具类** — `MessageUtil`、`PluginManager`
4. **注册权限码** — `PermCodeManager.init()`
5. **初始化数据库** — `HibernateUtil.init()`
6. **初始化各 Manager**：
   - `EconomyUtil` — 经济核心
   - `LotteryManager` — 彩票
   - `FishManager` — 钓鱼
   - `BankManager` — 银行
   - `TitleManager` — 称号
   - `PrivateBankManager` — 私人银行
   - `YiYanManager` — 一言
   - `GamesManager` — 游戏
   - `FactorManager` — 因子
   - `PluginPropsManager` — 道具
   - `TitleTemplateManager` — 称号模板
7. **注册消息监听** — `AuthorizeServer.registerEvents()` 扫描 `cn.chahuyun.economy.action` 包
8. **注册事件监听** — `SignEvent`、`FishStartEvent`、`FishRollEvent`

### 6.3 `onDisable` 阶段

1. 设置 `PLUGIN_STATUS = false`
2. 关闭 `GamesManager`
3. 关闭 `LotteryManager`
4. 关闭 `YiYanManager`
5. 停止 `HuYanScheduler`

## 八、注意事项

1. **ClassLoader 冲突**: mirai-console 的 app classloader 已含 log4j-api，插件不得再打包 log4j，否则触发 `LinkageError`
2. **Hibernate 冲突**: 不要将 Hibernate/HikariCP 打入插件 jar，运行期由 mirai-hibernate-plugin 提供
3. **版本迁移**: 部分版本升级需执行 `hye repair` 指令修复数据库
4. **KSP 生成代码**: `build/generated/ksp/` 下有 KSP 生成的 `AuthorizeRegistrar`，不要手动修改
5. **单 Bot 限制**: 插件仅支持绑定一个 Bot（由配置指定 Bot QQ 号）
6. **数据库支持**: H2 / SQLite / MySQL，通过配置切换
