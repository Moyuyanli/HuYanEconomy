# 私人银行系统实现说明

本文档说明当前代码中已经落地的私人银行功能与实现原理。`plan/v2.0.0/私人银行系统.md` 是设计目标，本文以当前源码实现为准，并在末尾列出与设计目标不一致或尚未完成的部分。

## 一、已实现功能

### 1. 银行创建与基础信息

- `银行创建 <code> <name>`：用户主银行存款达到 100,000,000 后，可以创建一家私人银行。
- 一个 QQ 只能创建一家银行。
- 创建时从用户主银行扣除 100,000,000，按 80/20 拆分：
  - 80% 进入该银行的准备金池。
  - 20% 进入该银行的流动金池。
- 新银行默认 3 星，默认存款利率跟随主银行基准利率。
- `银行描述修改`：行长在群内发送指令后，下一条消息会写入银行描述。
- `银行利率变更 <rate|max|min|now>`：行长调整储户存款利率，范围是主银行基准利率上下 0.3%。

### 2. 银行查询

- `银行列表`：列出最多 15 家银行，按星级排序。
- `银行信息 [code/name]`：渲染银行信息图片。
- 不带 `code/name` 时，优先展示发送者自己创建的银行；如果没有自建银行，再使用该用户记录的默认银行。
- `默认银行`：查询当前默认银行；未配置时等同主银行。
- `默认银行设置 <code/name>`：主动设置默认私人银行。
- `默认银行设置 main` / `默认银行设置 主银行`：切回主银行。
- 银行信息图展示：
  - 银行概况：行长、星级、平均评分、存款利率、存款总额、取款成功率、失信状态、银行描述。
  - 资金位置：准备金池、流动金池、放贷库存、风险保证金、国卷持仓、狐卷锁仓。
  - 额度与放贷：可借额度、发布总额、待收本息、当前一个可用标的。

### 3. 私人银行存取款

- `存款 <金额> [code/name]`：带 `code/name` 时向指定私人银行存款；不指定时存入主银行；行长不能向自己的银行存款。
- `取款 <金额> [code/name]`：带 `code/name` 时从指定私人银行取款；不指定时从主银行取款。
- `存款!` / `取款!`：默认银行快捷操作；未设置默认银行时落回主银行。
- 默认银行只由 `默认银行设置 <code/name|main|主银行>` 主动切换，不再由存取款、评分、贷款等成功操作自动写入。
- 存款时同样按 80/20 进入准备金池与流动金池。
- 取款按以下资金顺序尝试兑付：
  - 流动金池。
  - 放贷库存。
  - 行长钱包。
  - 可强制赎回的国卷。
  - 风险保证金。
- 取款失败会增加失败次数，并使银行进入 30 天失信期；失信期间利率回落到主银行基准利率。

### 4. 评分与星级

- `银行评分 <1-5> [描述] [code/name]`：储户评分。
- 评分要求用户在该银行有存款。
- 每个用户对同一家银行只能评分一次。
- 星级由三部分综合刷新：
  - 存款规模。
  - 取款成功率。
  - 平均评分。
- 星级会限制可同时发布的放贷标的数量。

### 5. 放贷、贷款与还款

- `放贷 <金额> [利率]` 或 `银行放贷 <金额> [利率]`：行长发布贷款标的。
- 金额支持普通数字和缩写：`100000`、`20K`、`10M`、`1G`、`1万`、`1亿`。
- 不填利率时，默认使用该银行当前存款利率。
- 贷款日利率范围固定为 1.0% - 18.0%。
- `贷款列表` / `放贷列表` / `贷款额度`：行长查看自己银行所有贷款额度、剩余额度、总额、日利率、期限和状态。
- `贷款利息修改 <offerId> <利率>` / `放贷利息修改 <offerId> <利率>`：行长修改尚可用贷款额度的日利率。
- `撤贷 <offerId>` / `贷款撤回 <offerId>` / `放贷撤回 <offerId>`：撤回尚未借出的剩余额度，剩余金额从放贷库存回流到流动金池；已借出的借款单不受影响。
- 发布放贷时，从银行流动金池扣除对应金额，冻结到放贷库存池，并创建一笔 `PrivateBankLoanOffer`。
- 星级越高，可同时启用的放贷标的越多：
  - 1 星最多 1 笔。
  - 5 星最多 5 笔。
- `贷款 <金额> [code/name]`：用户从银行贷款，系统自动选择满足金额且利率最低的启用标的。
- 贷款成功后：
  - 从放贷库存扣除贷款本金。
  - 同额进入银行流动金池。
  - 用户在该银行的存款本金增加。
  - 生成 14 天期限的借款单，借款时直接计算应还总额。
- `还款 <金额> [code/name]`：按创建时间顺序偿还该银行未结清借款。
- 还款优先从用户钱包扣，不足时再从主银行扣。

### 6. 国卷

- 系统每周确保生成一期国卷。
- `国卷` / `国卷查看` / `国卷列表`：查看本周国卷额度、倍率、锁仓天数，以及当前行长银行持仓。
- `国卷购买 <金额>`：行长用流动金池购买本周国卷。
- `国卷赎回 [持仓ID]`：赎回指定持仓；不带 ID 时尝试赎回全部未赎回持仓。
- 到期赎回按主银行基准利率和倍率计算收益；提前赎回按 90% 本金折价。

### 7. 狐卷

- 狐卷在每月 1 日、15 日发行。
- 每个发行日生成：
  - 3 张小额狐卷。
  - 6 张中额狐卷。
  - 1 张大额狐卷。
- `狐卷`：查看当前可竞标狐卷。
- `狐卷竞标 <code> <溢价金额> <接受利息>`：行长提交竞标。
- 同一家银行对同一狐卷重复竞标时会覆盖旧报价。
- 竞标要求：
  - 银行不能处于失信期。
  - 接受利息不能高于狐卷原始利率。
  - 接受利息不能低于主银行基准利率。
  - 准备金池要覆盖狐卷面额加溢价。
- 截标后按综合评分选择赢家：
  - 溢价。
  - 让利程度。
  - 银行星级。
  - 准备金规模。
- 中标后从准备金池锁定狐卷本金，溢价直接扣除；到期后本金与收益回到准备金池。

### 8. 定时任务

`PrivateBankManager` 初始化以下任务：

- 每天 04:10：私人银行储户利息结算，并刷新银行评级。
- 每天 04:20：追缴到期贷款。
- 每周一 00:05：确保本周国卷存在。
- 每月 1 日、15 日 08:00：发行狐卷。
- 每月 1 日、15 日 18:05：结算狐卷竞标。
- 每天 04:15：赎回到期狐卷持仓。

## 二、核心实现原理

### 1. 分层

- `economy-main/action`：Mirai 指令入口，例如 `PrivateBankAction`、`FoxBondAction`。
- `economy-core/usecase`：指令编排，例如 `PrivateBankUsecase`、`BankUsecase`、`FoxBondUsecase`。
- `economy-core/privatebank`：私人银行核心业务，例如 `PrivateBankService`、`PrivateBankFoxBondService`。
- `economy-data/data/repository`：数据访问，例如 `PrivateBankRepository`。
- `economy-data/model/privatebank`：DTO。
- `economy-image`：银行信息图片渲染。

### 2. 资金池

私人银行没有独立 DI 容器，业务通过 Kotlin `object` 单例协作。资金池通过银行 code 和固定 description 区分：

- `pb-reserve`：准备金池，存放在全局主银行账户路径。
- `pb-liquidity`：流动金池，存放在插件自定义账户路径。
- `pb-inventory`：放贷库存池，发布贷款时冻结资金。
- `pb-guarantee`：风险保证金池。
- `pb-fox-lock`：狐卷锁仓池。

业务层只通过 `EconomyUtil` 进行转账、扣款和入账，不直接操作余额表。

### 3. 数据对象

主要对象包括：

- `PrivateBankDto`：银行主体信息。
- `PrivateBankDepositDto`：储户在某银行的本金。
- `PrivateBankReviewDto`：评分记录。
- `PrivateBankLoanOfferDto`：行长发布的放贷标的。
- `PrivateBankLoanDto`：用户借款单。
- `PrivateBankGovBondIssueDto` / `PrivateBankGovBondHoldingDto`：国卷发行和持仓。
- `PrivateBankFoxBondDto` / `PrivateBankFoxBondBidDto` / `PrivateBankFoxBondHoldingDto`：狐卷、竞标和持仓。

Repository 会按 `DataSourceStrategyImpl.getVersion("privatebank")` 在 v1 实体和 v2 实体之间切换。

## 三、与 2.0.0 设计目标的差异

### 已按设计实现或基本实现

- 银行创建、银行列表、银行信息。
- 80/20 准备金和流动金模型。
- 银行描述修改。
- 利率变更，支持 `rate/max/min/now`。
- 存款、取款、评分。
- 放贷、贷款、还款。
- 放贷数量和利率上限与星级关联。
- 国卷购买、持仓、赎回。
- 狐卷竞标与多维评分。
- 失信状态和取款失败统计。
- 定时结息、定时追缴、债券发行与结算。

### 当前实现与设计目标不同

- 早期设计草案曾写创建门槛 10M；当前实现以 100M 为正确基准。
- 当前创建时扣 100M，并按 80M 准备金、20M 流动金拆分；风险保证金池目前不是创建时单独锁定的 10M。
- 设计文档提到 Logo、自定义 VIP 准入名单、称号位等玩法，当前 DTO 有 `vipOnly` / `vipWhitelist` 字段，但没有完整用户指令与流程。
- 设计文档中的“行长被动赚取利差”没有独立结算为行长收益；当前是直接给储户本金按银行利率滚存。
- 设计文档中的“行长全服公告通报”“连续一个月补足库存后恢复”等规则未完整实现；当前失信期主要通过时间戳、利率回落和部分操作限制体现。
- 国卷设计里提到竞标，但当前国卷实现更接近每周额度购买；多维竞标主要落在狐卷系统里。

### 需要注意的源码现状

- 国卷入口现在只负责国卷查看、购买、赎回；狐卷查看和竞标入口保留在私人银行指令中。
- 借款成功后，贷款本金会增加到用户在该银行的存款本金中，而不是直接进入用户钱包。
- 默认银行只由 `默认银行设置` 指令写入 `UserInfoDto.defaultPrivateBankCode`；普通存取款不会自动切换默认银行。
